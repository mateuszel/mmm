# Rapid MVP

Read `docs/OFFICIAL_CASE.md`, `docs/CURRENT_CONCEPT.md`, `docs/OPEN_DECISIONS.md`, `docs/ARCHITECTURE.md`, and `docs/HACKATHON_RUNBOOK.md`. Stop if the status is still `Discovery`. After the team explicitly approves implementation, define one observable happy path from user intent through availability/condition/seller analysis and negotiation to an explainable proposal. Specify deterministic fixtures, required model behavior, approval boundaries, fallback, and three-minute demo beats. Create tasks ordered by dependency and demo value; defer authentication, databases, real messaging, payments, and nonessential breadth.
