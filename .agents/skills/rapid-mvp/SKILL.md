# Rapid MVP

Read `docs/CURRENT_CHALLENGE.md`, `docs/ARCHITECTURE.md`, and `docs/HACKATHON_RUNBOOK.md`. Define one observable happy path from input to measurable result, its fixture data, required OpenAI behavior, approval boundary, and fallback. Create implementation tasks ordered by dependency and demo value. Prefer modifying the existing control plane and adapters over adding systems. Defer authentication, databases, and nonessential breadth. Preserve deterministic demo mode.
