# Challenge intake

Read `docs/CHALLENGE_BRIEF_TEMPLATE.md`, `docs/CURRENT_CHALLENGE.md`, and `docs/IDEA_BANK.md`. Extract the user, pain, outcome, constraints, available data, required integrations, and judging opportunity. Keep a table with separate “confirmed” and “assumption” columns. Produce exactly three product directions, score each against the repository weights, show the math, and recommend one without hiding uncertainty. Update `docs/CURRENT_CHALLENGE.md`; never invent partner capabilities.
