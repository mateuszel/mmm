# Hackathon runbook

| Time | Outcome |
| --- | --- |
| 10:30–10:50 | Process official challenge; mark facts vs assumptions |
| 10:50–11:10 | Score concepts and freeze one vertical slice |
| 11:10–13:00 | Working happy path end to end |
| 13:00–15:30 | Core logic and partner differentiation |
| 15:30–16:30 | Integration, tests, error handling |
| 16:30 | Feature freeze |
| 16:30–18:00 | Reliability, UI polish, pitch, fallback recording |

At briefing: complete `CURRENT_CHALLENGE.md`, run the challenge-intake skill, score options in `IDEA_BANK.md`, choose one, then run rapid-mvp. At freeze: run `pnpm demo:reset && pnpm check && pnpm test:e2e`, rehearse `DEMO_SCRIPT.md`, and record a fallback walkthrough.
