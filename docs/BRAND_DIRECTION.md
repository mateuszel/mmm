# Brand direction

**Status: Provisional.** Approved for the scripted prototype, not trademark-cleared for launch.

## Naming shortlist

| Candidate | Polish / English pronunciation | Trust and action signal | Icon potential | Collision risk | Score |
| --- | ---: | ---: | ---: | ---: | ---: |
| **Relyo** | 19/20 | 19/20 | 18/20 | 14/20 | **88/100** |
| Veyra | 18/20 | 15/20 | 18/20 | 8/20 | 77/100 |
| Trego | 18/20 | 17/20 | 17/20 | 5/20 | 75/100 |

`Relyo` is the provisional winner: short, pronounceable, consumer-facing and rooted in reliability without implying a bank or payment licence. A quick public search found an unrelated planning product using the same name, so this is a hackathon name only until a proper domain and trademark review. `Veyra` overlaps with active shopping and travel products. `Trego` overlaps directly with an AI ordering assistant and other apps.

## Brand idea

**Promise:** *Find the deal. Keep the boundary.*

Relyo is not a search-results page. It is a delegated shopping agent that carries evidence to a decision and stops when the user’s boundary is crossed.

### Recognisable motif: protected handoff

Two strong paths converge on a verified point. The motif represents evidence and user intent meeting at a controlled action boundary.

- Search: open paths move toward a result.
- Mandate: paths lock at a verified point.
- Source cards: a thin path carries source evidence into a decision.
- Seller communication: messages move along a protected channel.
- Call: the path pulses while the fictional relay is active.
- Risk: the path breaks before an unsafe handoff.
- Receipt: the path closes with a verified dot.

## Visual system

| Token | Value | Use |
| --- | --- | --- |
| Warm canvas | `#F4F2EC` | Product background |
| Paper | `#FCFBF7` | Source and conversation surfaces |
| Ink | `#121712` | Type and 2–3 px structure |
| Trust green | `#123F33` | Active mandate and protected actions |
| Signal lime | `#D6FA52` | One verified point, never a large fill |
| Muted red | `#B33A32` | Confirmed risk and rejection only |
| Hairline grey | `#CBD0C9` | Secondary structure |

- Typeface: native grotesk sans stack; no remote font dependency.
- Borders: 2 px default, 3 px for primary structure.
- Corners: 6–12 px; never pill-shaped for large surfaces.
- Shadow: none by default; evidence may use a single low-opacity offset.
- Gradients, glass, glow, decorative blobs and oversized dashboard cards are prohibited.
